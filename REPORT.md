# Отчёт — ЛР №2: обесцвечивание и бинаризация

## Цель

1) Привести полноцветное (RGB) растровое изображение к полутоновому (1 канал) через взвешенное усреднение каналов.  
2) Выполнить адаптивную бинаризацию полутонового изображения методом пороговой обработки с усреднением по окну.

## Формулы

### 1) Полутон

Яркость пикселя:

`Y = 0.299 * R + 0.587 * G + 0.114 * B`

Результат сохраняется как BMP с 1 яркостным каналом (`*_gray.bmp`).

### 2) Адаптивная пороговая обработка (локальное среднее)

Для каждого пикселя берётся окно `window x window` (на границах окно обрезается).  
Порог:

`T(x,y) = mean(window) - offset`

Бинаризация:

- если `Y(x,y) >= T(x,y)` → 255 (белый)
- иначе → 0 (чёрный)

## Реализация

Файл: `lab2.py`

- Полутон: функция `to_grayscale(...)` (без `convert('L')`).
- Локальное среднее: через интегральное изображение (summed-area table) для ускорения.
- Бинаризация: `adaptive_threshold_mean(...)`.

## Демонстрация (до/после)

Скрин/картинка сравнения сохраняется автоматически:

- `output/*_compare.png` (оригинал | полутон | бинарное)

Вставьте сюда 5–6 примеров (по требованию задания), например:

- `output/sample_contours_compare.png`
- `output/sample_xray_compare.png`
- `output/sample_cartoon_compare.png`
- `output/sample_fingerprint_compare.png`
- `output/sample_text_uneven_compare.png`

## Запуск

```powershell
python -m pip install -r requirements.txt
python .\lab2.py --generate-samples
python .\lab2.py --window 31 --offset 5
```

## Подбор параметров

- Увеличение `window` сглаживает локальные изменения и делает порог «более глобальным».
- Увеличение `offset` обычно даёт больше белых пикселей (полезно для тёмного текста на светлом фоне).
